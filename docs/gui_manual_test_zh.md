# GUI 手动测试清单

本清单用于开发机上手动验收 PyQt6 GUI。测试前确认已经完成 `pku3b init`，并存在 `secrets/api_keys/deepseek_key.txt`。

## 1. 启动在线 GUI

如果当前网络需要代理，先在同一个终端里设置临时代理：

```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
export all_proxy=http://127.0.0.1:7897
```

然后在仓库根目录启动：

```bash
.venv/bin/python -m src --online
```

预期结果：

- 窗口标题为 `PKU Captain`。
- 左侧是仪表盘，右侧是对话；工具调用会融合显示在对话流中。
- 整体视觉为北大红 / 白 / 浅灰底色，主按钮为北大红。
- 对话面板顶部出现系统提示，说明 GUI 已启动。
- 窗口底部状态栏显示当前状态。

## 2. 测试仪表盘刷新

如果刷新课表时需要手机令牌，在仪表盘右上角 `课表 OTP` 输入框填入当次 OTP，再点击刷新。该输入不会保存。

点击左侧仪表盘的“刷新”按钮。

预期结果：

- 按钮短暂显示“刷新中”。
- 如果提供了有效 OTP，今日课表卡片显示当天课程；当天无课时显示完整课表预览。
- DDL 卡片显示未来作业。
- 通知卡片显示课程通知。
- 天气卡片显示北京大学天气。
- 讲座卡片显示讲座推荐或暂无讲座。
- 树洞卡片显示新消息状态。
- P-Lib 卡片显示下载额度，并提供搜索 / 登录入口。
- “最近刷新”时间更新。

## 3. 测试今日简报

点击“今日简报”按钮。

预期结果：

- 按钮短暂显示“生成中”。
- 对话面板出现“正在生成今日简报...”。
- 随后出现包含天气、今日截止、课程公告、近期讲座的简报。

## 4. 测试 Agent 对话和工具调用

在输入框输入：

```text
今天和最近有什么作业？
```

按 Enter 发送。

预期结果：

- 用户消息出现在对话面板。
- 发送按钮显示“处理中”。
- 对话流中出现工具调用过程。
- 如果调用 `pku3b_assignments`，工具调用块应显示作业摘要，而不是整段 JSON。
- 对话面板最终出现 Agent 回答。

## 5. 测试课程通知和讲座

- 在课程通知卡片点击一条通知。
- 预期弹出完整通知详情，包含课程、发布时间、正文。
- 在讲座推荐卡片点击一条讲座。
- 预期弹出讲座详情，包含时间、地点、主讲人和链接。
- 点击讲座卡片的“筛选”，输入关键词或日期范围。
- 预期筛选结果在弹窗中刷新。

## 6. 测试 P-Lib 和树洞入口

- 点击 P-Lib 卡片的“登录”，输入 P-Lib 账号后登录。
- 点击 P-Lib 卡片的“搜索资料”，搜索课程资料并查看详情。
- 点击页眉“树洞”，检查树洞消息列表和登录 / 短信验证码区域。

## 7. 测试提醒、记忆和知识库入口

- 点击页眉“提醒”，新增一条 ISO 时间格式提醒，例如 `2026-06-02T18:00:00`。
- 预期提醒出现在列表中，并可标记完成或删除。
- 点击页眉“记忆”，新增一条 key/value 记忆，刷新后仍可看到。
- 如果用 `--rag` 启动，点击“知识库”并输入关键词，预期显示检索片段。
- 如果未用 `--rag` 启动，点击“知识库”应提示知识库检索未开启。

## 8. 测试快捷键

- 输入一行文字后按 Enter：应发送消息。
- 按 Shift+Enter：应在输入框内换行，不发送。

## 9. 离线兜底测试

关闭窗口后运行：

```bash
.venv/bin/python -m src
```

预期结果：

- GUI 正常启动。
- 仪表盘中在线工具显示不可用状态。
- 对话侧栏仍可用 EchoLLMProvider 回显。
